## 💡 Feature Prioritization Framework

### How We Decide What to Build

**Framework: RICE Scoring**
```
(Reach × Impact × Confidence) / Effort = Score

Reach: How many users does this affect?
Impact: How much does it change their success?
Confidence: How sure are we this is the right solution?
Effort: How many weeks of engineering?
```

**Example 1: GitHub Integration**
```
Reach: 10,000 users (high)
Impact: 50% time savings (massive)
Confidence: 90% (proven pattern)
Effort: 2 weeks
Score: (10K × 3 × 0.9) / 2 = 13,500
→ Build immediately
```

**Example 2: VSCode Extension**
```
Reach: 5,000 users (medium)
Impact: 30% time savings (high)
Confidence: 70% (uncertain UX)
Effort: 4 weeks
Score: (5K × 2.5 × 0.7) / 4 = 2,187
→ Build in Q2
```

**Example 3: Custom Domain Support**
```
Reach: 100 users (low)
Impact: 10% improvement (low)
Confidence: 100% (trivial)
Effort: 1 week
Score: (100 × 1 × 1) / 1 = 100
→ Build only if has other dependencies
```
